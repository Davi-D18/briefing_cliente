# Briefing Cliente - Painel de Usuários

Este repositório contém a documentação, o planejamento e os protótipos de um painel para visualização de usuários, posts e comentários, consumindo a API [JSONPlaceholder](https://jsonplaceholder.typicode.com).

O diferencial deste projeto é o foco na **comunicação acessível** e no **tratamento humanizado de erros**, garantindo que usuários não técnicos compreendam o que está acontecendo no sistema, mesmo diante de falhas.

## 🚀 Sobre o Projeto

O projeto foi desenvolvido como parte de um dos desafios na Labs para atender a uma necessidade de monitoramento de usuários de forma simples e intuitiva. O sistema permite:
1.  **Listar Usuários:** Ver uma visão geral de todos os usuários cadastrados.
2.  **Explorar Posts:** Acessar as publicações realizadas por um usuário específico.
3.  **Feedback Humanizado:** Receber orientações claras em vez de códigos de erro técnicos.

## 📁 Estrutura do Repositório

O repositório está organizado de forma evolutiva, dividido por entregas semanais:

### 📅 [Semana 1](./semana-1/) - Fundação e Planejamento
Focada na análise do briefing e mapeamento técnico inicial.
*   **`exploracao_api/`**: Documentação detalhada de como os dados estão estruturados na API (usuários, posts e comentários).
*   **`proposta-cliente/`**: Proposta comercial e técnica escrita em linguagem simples para o cliente final.
*   **`qualidade-integração/`**: Registro de revisões e perguntas estratégicas para alinhar expectativas.
*   **`tradutor_tecnico/`**: Mapeamento das ações do usuário para rotas e métodos da API.

### 📅 [Semana 2](./semana-2/) - Resiliência e Experiência do Usuário
Focada no tratamento de erros e na interface de uso.
*   **`tradutor_tecnico/`**: Contém a evolução técnica do projeto:
    *   `index.html`: Protótipo visual do painel.
    *   `mensagens_amigaveis.md`: Guia de tradução de erros técnicos para mensagens compreensíveis.
    *   `log.md`: Estrutura de logs para suporte técnico.
    *   `tradutor_tecnico.md`: Versão atualizada do guia técnico com foco em estados de erro.

## 🛠️ Tecnologias Utilizadas
*   **API Externa:** [JSONPlaceholder](https://jsonplaceholder.typicode.com)
*   **Documentação:** Markdown (clara, organizada e versionada)
*   **Interface:** HTML/CSS para prototipagem rápida
